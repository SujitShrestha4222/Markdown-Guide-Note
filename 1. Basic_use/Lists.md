# 📋 Lists in Markdown:
Lists help organize content clearly.



---



## ✅ Unordered List:
Use `-`, `*`, or `+`:

```md
- Item 1
- Item 2
  - Subitem 2a
  - Subitem 2b
* Item 3
+ Item 4
```

* Item 1
* Item 2
  * Subitem 2a
  * Subitem 2b
- Item 3
* Item 4



---



## ✅ Ordered List:
```md
1. First
2. Second
   1. Subitem 2a
   2. Subitem 2b
3. Third
```

1. First
2. Second

   1. Subitem 2a
   2. Subitem 2b
3. Third



---



## ✅ Mixed Lists:
You can mix ordered & unordered:

```md
1. Fruits
   - Apple
   - Banana
2. Vegetables
   - Carrot
   - Spinach
```

1. Fruits
   * Apple
   * Banana
2. Vegetables
   * Carrot
   * Spinach



---



## 💡 Tips:
* Use **2 spaces** for nested lists (makes GitHub render correctly)
* Keep numbering simple — GitHub auto-corrects `1.` `1.` `1.`



---



## 🎯 Real Use Case:
Great for:
* ✅ Features
* ✅ Steps to install
* ✅ To-do lists
